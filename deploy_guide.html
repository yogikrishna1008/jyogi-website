Deploying api.py to Railway

From your local files to a live URL in under 10 minutes.

✦ What You're Deploying

Four new files sit alongside your existing jyogi/ package — the package itself is unchanged.

jyogi-website/              ← your GitHub repo
│
├── api.py              ← NEW: FastAPI backend (the main file)
├── Procfile            ← NEW: tells Railway how to start the server
├── railway.json        ← NEW: Railway config (health check, restart policy)
├── requirements.txt    ← UPDATED: adds fastapi + uvicorn
│
├── app.py                  ← UNCHANGED: your Streamlit app
├── jyogi/                  ← UNCHANGED: entire package
│   ├── calc_engine.py
│   ├── pipelines/
│   ├── engines/
│   └── ...

✦ Step-by-Step Deployment

1
Push your files to GitHub
Copy the 4 new files into your repo root, then push. Railway deploys directly from your GitHub repo.
git add api.py Procfile railway.json requirements.txt
git commit -m "add FastAPI backend"
git push
2
Create a Railway project
Go to railway.app → New Project → Deploy from GitHub Repo → select your jyogi-website repo. Railway auto-detects Python and installs everything.
Free tier: Railway gives 500 hours/month free — enough for ~16 hours/day of uptime. Your astrology API will never run 24/7, so this is plenty.
3
Add environment variables
In Railway → your project → Variables tab, add these:
VariableValue
OPENAI_API_KEYYour OpenAI key — sk-proj-...Required
PORTRailway sets this automaticallyAuto-set
Security: Never put your API key in code or GitHub. Railway's Variables are encrypted at rest.
4
Get your Railway URL
After deploy succeeds: Railway → Settings → Domains → Generate Domain. You'll get something like:
https://jyogi-api-production.up.railway.app
Test it by visiting /health in your browser — you should see:
{"status": "ok", "service": "Jyogi AI API", "version": "1.0.0"}
5
Update the URL in jyogi_website.html
Find this line in your website and replace with your Railway URL:
// Find this in jyogi_website.html:
const resp = await fetch('https://jyogi-api.railway.app/api/chart'

// Replace with your actual Railway URL:
const resp = await fetch('https://jyogi-api-production.up.railway.app/api/chart'
6
Update CORS to your Netlify URL
In api.py, update the ALLOWED_ORIGINS list with your actual Netlify domain:
# In api.py — find ALLOWED_ORIGINS and update:
ALLOWED_ORIGINS = [
    "https://YOUR-SITE.netlify.app",    ← your actual Netlify URL
    "https://www.jyogi.com",            ← your custom domain (if any)
    "http://localhost:3000",
]

✦ API Endpoints Reference

GET /health Check if API is running
Returns {"status":"ok"} — use this as your Railway health check URL.
POST /api/geocode City name → coordinates 
// Request body:
{ "city": "Mumbai" }

// Response:
{ "lat": 19.0760, "lon": 72.8777, "address": "Mumbai, Maharashtra, India" }
POST /api/chart Full Vedic birth chart 
// Request body:
{
  "name":     "Priya Sharma",
  "dob":      "1992-04-15",         // YYYY-MM-DD
  "time":     "07:30",              // HH:MM 24h (optional, defaults 12:00)
  "city":     "Delhi",
  "question": "When will I get married?"  // optional
}

// Response includes:
// lp_num, lp_title — Life Path number
// chart_text       — Full sidereal chart positions
// lagna_text       — Lagna & personality analysis
// dasha_text       — Current Mahadasha interpretation
// ss_text          — Sadhe Sati status
// qa_text          — AI answer to question (if provided)
// pdf_id           — UUID to download PDF: /api/pdf/{pdf_id}
// sections         — [{heading, text}, ...] for display
POST /api/tarot AI tarot reading 
// Request body:
{
  "spread_id":   "3",                      // "1" / "3" / "celtic" / etc
  "question":    "What does my career hold?",
  "client_name": "Rahul"                    // optional, for PDF only
}

// Response includes:
// spread_name   — human-readable spread name
// card_summary  — cards drawn with orientations
// affirmations  — per-card affirmations
// ai_answer     — Jyogi's AI interpretation
// pdf_id        — UUID to download PDF
GET /api/pdf/{pdf_id} Download generated PDF
Returns the PDF file. The pdf_id comes from the chart or tarot response. PDFs are stored in memory and cleared on server restart — download immediately after generating.

✦ Rate Limiting & Security

Built-in protection: 20 AI calls per IP per hour. This mirrors your original Streamlit session limit but works across sessions and restarts. No database needed — it resets on deploy (which is fine for your usage pattern).

The geocode and PDF download endpoints do not consume rate limit credits — only /api/chart and /api/tarot count (they make OpenAI calls). The geocoder results are also cached in memory to avoid repeat calls for the same city.

✦ Testing Locally Before Deploy

# Install dependencies
pip install fastapi uvicorn openai pyswisseph geopy reportlab

# Set your API key
export OPENAI_API_KEY=sk-proj-your-key-here

# Start the server
uvicorn api:app --reload --port 8000

# Test health check
curl http://localhost:8000/health

# Test geocode
curl -X POST http://localhost:8000/api/geocode \
  -H "Content-Type: application/json" \
  -d '{"city": "Mumbai"}'

# Test chart (no question = pure Swiss Ephemeris, no AI cost)
curl -X POST http://localhost:8000/api/chart \
  -H "Content-Type: application/json" \
  -d '{"name":"Test","dob":"1990-01-01","time":"12:00","city":"Delhi"}'
Interactive docs: With the server running, visit http://localhost:8000/docs for a full Swagger UI where you can test every endpoint visually.

✦ Common Issues

!
pyswisseph build fails on Railway
Railway's Nixpacks builder handles C extension compilation automatically. If it fails, add a nixpacks.toml file:
# nixpacks.toml (create in repo root if needed)
[phases.setup]
nixPkgs = ["gcc", "python3"]
!
CORS error in browser ("blocked by CORS policy")
Your Netlify URL isn't in ALLOWED_ORIGINS in api.py. Update the list and redeploy:
ALLOWED_ORIGINS = [
  "https://YOUR-EXACT-NETLIFY-URL.netlify.app",
  ...
]
!
OpenAI API key not working
Check Railway → Variables → OPENAI_API_KEY is set exactly (no spaces, no quotes). Then trigger a redeploy from Railway dashboard.
!
City not found (geocode returns 404)
The geocoder uses OpenStreetMap Nominatim. Some small Indian towns need the state added: "Shirdi, Maharashtra" instead of just "Shirdi". The website's astrology form sends a graceful WhatsApp fallback if geocoding fails.
